# subway side project
지하철 노선/역/구간을 관리하고, 두 역 사이 최단 시간 경로를 조회하는 백엔드 프로젝트입니다.

## 1. 주요 기능
- 노선(Line) 등록/수정/삭제/조회
- 역(Station) 등록/수정/삭제/조회
- 구간(Segment) 관리 (노선 내 역 연결)
- 노선별 역 목록 조회 (정렬 보장)
- 경로 탐색: 출발역 → 도착역 최단 시간 경로 조회

## 2. 적용 기술 & 설계 포인트
### 2-1. 조회 최적화
- 역, 노선명을 그대로 저장하나 조회시엔 대소문자 구분없이 확인하여 중복 저장 방지
- 변화가 적고 자주 조회되는 데이터(예: 노선별 역 목록, 경로 탐색 결과)는 Redis 캐시로 최적화
- 캐시 전략
  - Cache Aside: 조회 시 캐시 미스면 DB 조회 후 캐시 저장
- 다익스트라 + 우선순위 큐로 최단 시간 경로 계산
### 2-2. 버전 관리를 통한 동시 요청 제어
- station, line 수정 요청은 버전 관리를 통해 동시성 제어
- seg 수정 요청은 모두 line을 호출하여 동시 요청 관리
### 2-3. 책임 분리
- 프로젝트는 core(domain), app, db 모듈로 분리
- domain은 도메인 생성 규칙, db는 mysql 연결 관련, app은 boot와 web 관련 포트 및 서비스 담당
- db 모듈에 Test Fixture 모듈을 만들어 app 모듈에서도 쉽게 test 공통 코드를 사용

## 3. 아쉬운 점
### 3-1 도메인 모듈의 의미 퇴색
- 도메인과 db로 분리하는 과정에서 엔티티와 도메인을 변환하는 과정을 줄이려 하다보니 app 레이어의 역할이 많아지고, 도메인은 생성 및 간단한 검증 역할만 가지게 되었습니다.
- 앞으론 port와 policy를 구체적으로 설계해 도메인 모듈이 규칙을 가져가게 하고자 합니다.
