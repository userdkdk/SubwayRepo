# subway side project
지하철 노선/역/구간을 관리하고, 두 역 사이 최단 시간 경로를 조회하는 백엔드 프로젝트입니다.
조회 성능을 위해 Redis 캐시를 적용했고, 책임 분리를 위해 멀티모듈 아키텍처로 설계했습니다.

## 1. 핵심 기능
- 노선(Line) 등록/수정/삭제/조회
- 역(Station) 등록/수정/삭제/조회
- 구간(Segment) 관리 (노선 내 역 연결)
- 노선별 역 목록 조회 (정렬 보장)
- 경로 탐색: 출발역 → 도착역 최단 시간 경로 조회
  - 우선순위 큐 기반 다익스트라(PQ) 적용
  - 현재 버전은 환승시간 미고려 (추후 확장 포인트로 분리)

## 2. 적용 기술 & 설계 포인트
### 2-1. 조회 최적화
- 역, 노선명을 그대로 저장하나 조회시엔 대소문자 구분없이 확인하여 중복 저장 방지
- 변화가 적고 자주 조회되는 데이터(예: 노선별 역 목록, 경로 탐색 결과)는 Redis 캐시로 최적화
- 캐시 전략
  - Cache Aside: 조회 시 캐시 미스면 DB 조회 후 캐시 저장 
### 2-2. 최단 경로 탐색 (PQ 다익스트라)
- 다익스트라 + 우선순위 큐로 최단 시간 경로 계산
- 제약/확장 포인트
  - 환승시간 미고려하여 이후 노선별 환승 비용 추가
### 2-3. 책임 분리
- 프로젝트는 domain과 app 두 모듈로 분리했습니다.
- domain lombok만 의존하고 있습니다.
- DB/Redis/Web은 app 내부에서 패키지 경계로 분리했습니다.
